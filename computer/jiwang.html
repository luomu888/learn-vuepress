    # 第一章 概述

    # 计算机在信息时代的作用

    不可或缺，通信基础设施发展为重要的信息服务基础设施

    # 因特网的概述

    # 网络、互联网(互连网)和因特网

    网络由若干结点和连接这些结点的链路组成

    多个网络可以由路由器互连，即互联网(网络的网络)

    因特网是世界上最大的互连网络

    internet(互联网)一个通用名词，网络之间的通信协议是任意的

    Internet(因特网)一个专用名词，采用TCP/IP协议族作为通信的规则

    # 因特网发展的三个阶段

    1969(从单个网络ARPANET向互联网发展)

    ===>1983(TCP/IP协议成为ARPANET的标准协议，即因特网诞生时间)

    ------》1985(形成三级结构的因特网，NSFNET(主干网、地区网和校园网))

    ------》1993(多层次ISP结构的因特网)

    ------》1994(万维网WWW技术促使因特网迅猛发展)

    ISP(因特网服务提供商)：例中国电信、中国联通、中国移动

    A用户连接B用户可能经过多个ISP，因为可能先连接到最底层的Internet,其次在访问B用户的ISP

    # 因特网的标准化工作

    特点：面向公众，因特网所以的RFC技术文档都可以下载

    因特网协会ISOC是一个国际性组织，对因特网进行全面管理

    因特网体系结构委员会IAB，负责有关协议的开发

    因特网工程部IETF，针对协议的开发和标准化

    因特网研究部IRTF，对问题进行开发

    制订因特网的正式标准(4个阶段)：

    因特网草案(不是PFC)---建议标准(开始是PFC)---草案标准---因特网标准(小部分的PFC成为因特网标准)

    # 因特网的组成

    边缘部分：由所有连接因特网上的主机组成，用户可以直接使用，用来资源共享

    核心部分：由大量网络和连接这些网络的路由器组成，为边缘部分提供服务(提供连通性和交换)

    # 三种交换方式

    # 电路交换(Circuit Switching)

    电话交换机接通电话线方式

    三个步骤：

    1、建立连接（分配通信资源）

    2、通话（一直占用通信资源）

    3、释放连接（归还通信资源）

    而计算机靠这个传送数据是，效率比较低，因为用户占用资源；来进行编辑和修改，不传送，这样资源就被空闲

    # 分组交换(Packet)

    发送方：构造分组(数据被拆分)、发送分组(构造时加个标记，让路由器好识别转发到哪)

    路由器(存储转发)：缓存分组、转发分组

    接收方：接收分组、还原报文(就是把数据组装)

    # 报文交换(Message)

    将一个很多的数据直接传送，没有拆分，所以存储点需要很大的空间

    # 电路、报文、分组的对比

    电路：优点：一条线，有序传输，控制简单；缺点：建立连接时间长，效率低，线路被占，只能等

    报文：优点：无需连接，效率高，线路被占，还可以使用其他的；缺点：转发有时延，提供较大的存储空间，需要传输格外的信息量

    分组：优点：在报文的基础上，简化存储，缺点：需要传输格外的信息量，让路由器知道怎么转发，还得还原

    # 计算机网络的定义和分类

    # 计算机网络的定义

    说的是计算机网络，和网络的组成不同

    最简单的定义：一些互相连接的、自治的计算机的集合

    互连：计算机之间可以进行数据通信

    自治：独立的计算机，有自己的硬件和软件，单独运行

    集合：至少两台计算机

    现在时代的定义：一些通用的、可编程的硬件互连而成，可编程的硬件(智能手表、手机、电脑)能够用来传送多种不同类型的数据，并支持多种应用(包括今后可能出现的各种应用)

    # 计算机网络的分类

    按交换技术：电路、报文、分组

    按使用者：公用网(交钱即可)、专用网(军队、火箭研究所)

    按传输介质：有线网络、无线网络

    按覆盖范围分类(WMLP)：广域网WAN(核心，国家级)、城域网MAN(城市)、局域网LAN(校园、公司)、个域网PAN(鼠标、键盘、耳机等)

    按拓扑结构分类：

    总线型网络：一条线，有许多连，一个崩坏，全坏

    星型网络：一个路由器，多人连，一个崩坏，不影响其他，除非坏的是路由器

    环形网络：连接形成一个环，和总线差不多，只是把线弄成环

    网状型网络(常用)：多个路由器

    # 计算机网络的性能指标

    速率：连接在计算机网络上的主机在数字道上传送比特的速率，即比特率或数据率 相当速度

    单位：bit/s(即b/s,bps) kb/s=10^3b/s Mb/s Gb Tb 进位10^3

    # 计算机性能指标（2）-时延带宽积、往返时间、利用率、丢包率

    时延带宽积：传播时延*带宽，圆柱的体积，链路的时延带宽积又称为以比特为单位的链路长度

    往返时间(RTT)：因特网上的信息不仅仅单方向传输，而是双向交互，双向来回时，耗时最多的是链路

    利用率：分为信道利用率(有几个数据通过)和网络利用率(全网络的信道利用率的加权平均)

    令D0表示网络空闲时的时延，D表示网络当前的时延，利用率U，关系：D=D0/1-U

    信道利用率并非越高越好，要控制在50%之下，因此要动态调整信道利用率，如果超过50%，那么就要增大线路的带宽

    丢包率：即分组丢失率，丢失的分组数量与总分组数量的比率

    分组丢失出现的原因主要有2种：传输过程出现误码；第二，分组交换机(路由器)已满，它就被丢弃，所以在通信量比较大时就可能造成网络拥塞

    # 计算机网络体系结构

    # 常见的计算机网络体系结构

    OSI体系结构(法律上的国际标准)：物理层、数据链路层、网络层、运输层、会话层、表示层、应用层

    TCP/IP体系结构(事实上的国际标准，具有一定的商业化)：网络接口层、网际层、运输层、应用层

    原理体系结构(适用于教学)：物理层、数据链路层、网络层(和网际层差不多)、运输层、应用层

    路由器(网络接口层、网际层)

    最重要的是网际层(IP)、使用IP协议互连不同的网络接口(有线的以太网接口、无线的局域网的WiFi接口等)、运输层(TCP、UDP)，TCP提供可靠服务给应用层(HTTP、SMTP)；UDP提供不可靠**(DNS、RTP)**

    # 分层的必要性

    将若大的问题通过分层解决，即局部处理

    物理层：解决传输01比特位，要接口、双绞线网线、0低电压1高电压；以太网接口(电脑连学校网线(双绞线网线)旁边那个)

    解决使用何种信号来传输比特的问题

    数据链路层：解决分组在一个网络(或一段链路)上的传输问题

    网络层：解决分组在多个网络上传输问题(路由)，IP地址前7为相同，说明连的是一个路由器

    运输层：解决进程之间基于网络的通信问题

    应用层：解决通过应用进程的交互来实现特定网络应用的问题

    支持万维网应用的HTTP协议

    电子邮件的SMTP协议

    文件传送的FTP协议

    # 分层思想举例

    在主机的浏览器输入地址，发起请求

    应用层：将它转换为HTTP请求报文

    运输层：添加TCP，变为TCP报文段

    网络层：添加IP，变为IP数据报

    链路层：添加ETH，变为帧

    物理层：添加前导码，解析时把前导码去掉，变为比特流，其次去掉ETH....

    # 专用术语

    实体：硬件或软件进程 对等实体：收发双方相同层中的实体，例应用层(发)和应用层(收)是对等实体

    协议：控制两个对等实体进行逻辑通信(虚的)的规则的集合，必须是一层一层来，不可能直接应用层到应用层

    协议的三要素：语法 语义 同步

    语法：定义交换信息的格式

    语义：定义收发双方要完成的操作

    同步：定义收发双方的时序关系，即一方已发送，一个是否已接收

    要实现本层协议(即逻辑通信)，必须使用下面一层所提供的服务
//协议水平的，服务垂直
享受服务，无需懂原理，即不知道它(下面一层)的协议

    服务访问点(上下的，即在一个体系结构)：相邻两层的实体交换信息的逻辑接口，区分不同的服务类型

    数据链路层：服务访问点为类型

    网络层：协议字段

    运输层：端口号

    服务原语：上使用下，交换一些命令，命令即服务原语

    协议数据单元PDU：对等层次之间传送的数据包

    比特流---帧---分组或IP数据报---TCP报文段或UDP用户数据段---报文

    服务数据单元SDU：同一个系统(即一个体系结构)内，层和层之间交换的数据包

    多个SDU可以合成一个PDU，一个SDU可划分为几个PDU

    # 第二章 物理层

    # 物理层的基本概念

    传输媒体：分为两种，导引型传输媒体，非导引型传输媒体

    导引型传输媒体：双绞线、同轴电缆、光纤

    非导引型传输媒体(自由空间)：微波通信（2~40GHz），例：WIFI：2.4G、5.0G

    物理层考虑的是怎样才能在连接各种计算机的传输媒体上传输数据比特流

    物理层协议的主要任务：

    机械特性：接线器的形状和尺寸、引脚数目和排列、固定和锁定装置

    电气特性：接口电缆的各条线上出现的电压范围

    功能特性：某条线的某一电平的电压表示何种意义

    过程特性：不同功能的各种可能事件的出现顺序

    # 物理层下面的传输媒体

    # 导引型传输媒体

    同轴电缆：硬件：内导体、绝缘层、屏蔽层、绝缘保护套层，从横切面可以看出同轴心，所以叫同轴电缆

    常见的同轴电缆：

    基带同轴电缆（50欧）：数字传输，过去用于局域网

    宽带同轴电缆（75欧）：模拟传输，目前用于有线电视

    # 信道的极限容量

    通信质量较差的信道：会出现失真

    失真元素：码元传输速率、信号传输距离、噪声干扰、传输媒体质量等

    奈氏准则(码元传输速率)：

    理想低通信道的最高码元传输速率（实际比它小）=2W 码元/秒 W：信道带宽（单位为Hz）

    理想带通信道……=W 码元/秒

    ==码元速率又称为==波特率、调制速率、波形速率或符号速率，它与比特率有一定关系

    当1个码元携带n比特的信息量时，波特率转换为比特率时**，数值乘以n**

    香农公式：高斯白噪声干扰的信道的极限信息传输速率

    c=W*log2(1+S/N)

    c：信道的极限信息传输速率

    W：信道带宽

    S：信道内所传信号的平均功率

    S/N：信噪比，单位（dB），计算公式为信噪比=10*log10(S/N)

    【2014年 题35】：不会影响信道数据传输速率的是（）

    答案：D；由香农公式知AB答案有影响，而C答案影响奈氏准则

    【2009年 题34】：在无噪声情况下，若某通信链路的带宽为3kHz，采用4个相位，每一个相位具有4种振幅的QAM调制技术，则该通信链路的最大数据传输速率是（）

    # 数据链路层概述

    数据链路：把实现通信协议的硬件和软件加到链路（一个结点到另一个结点）上

    可靠传输：对于不可靠的传输，出现误码直接丢弃即可，而可靠传输是发送方发送什么，接收方就能收到什么

    # 差错检测

    传输错误的比特占所传输比特总数的比率称为误码率BER

    帧检验序列FCS字段

    循环冗余校验CRC：

    收发双方约定一个生成多项式G(x)，发送方基于待发送数据和生成多项式计算差错检测码，添加到待传输数据一起传输，接收方通过生成多项式计算收到的数据是否产生误码，余数为0表示无误码

    例：待发送的信息为101001，生成多项式为G(x)=x^3+ x^2 +1，计算发送方余数

    解：1、构造被除数（即待发送的信息后面补零，补生成多项式最高次数个），101001000，最高次数为3，加3个零

    2、构造除数（生成多项式各项系数构成的比特串）即1101

    3、做除法

    4、余数的位数与生成多项式的最高次数相同，即有3位，前面补0 001

    检测码只能检测错误,不能定位错误,因此无法纠正错误,

    解决方案:检测重传,丢弃检测到差错的帧

    # 可靠传输

    # 可靠传输的基本概念

    数据链路层向上层提供的服务类型:

    不可靠传输服务(有线链路):丢弃误码的帧

    可靠传输服务**(无线链路**):实现发送端发送什么,接收端收到什么(即接收端把误码信息传回发送方,发送方进行重新发送)

    比特差错只是传输差错的一种,传输差错还包括分组丢失(即其中路由器已满,数据被丢弃).分组失序(先发送的分组后到达)以及分组重复(网络影响,发送方又发送一次,导致网络好了之前数据和现在重复)

    可靠传输服务并不仅局限于数据链路层,其他层均可选择实现可靠传输,可靠传输服务开销比较大,是否使用取决于应用需求

    例:

    网络层:IP向上传提供无连接,不可靠传输服务

    运输层:TCP向上层提供可靠传输服务,UDP提供不可靠传输服务

    # 可靠传输的实现机制—停止-等待协议（SW stop wait，也叫自动请求重传ARQ）

    确认与否定：发送方发送DATA，接收方接收，发ACK表示确认，如果DATA出现误码，接收方发送NAK，并把DATA丢弃，发送方重新发送DATA

    超时重传：DATA丢失，发送方接收不到ACK或NAK，发送方进行超时重传，重传时间选为略大于“从发送方到接收方的平均往返时间”

    确认丢失：即ACK丢失，需要给数据分组编号，即编号0和1，停止和等待，重传下次数据和上次数据一样，那么就丢弃现在传的

    确认迟到：即ACK可能重复，需要给ACK分组编号，也是0和1，注意的是数据链路层一般不会出现ACK分组迟到的情况

    信道利用率U=Td/Td+RTT+Ta

    Td即发送方的发送时延

    RTT即2倍传播时延（往返时间）

    Ta即接收方的发送时延，一般忽略不计，远小于发送方的发送时延，没有处理时延

    当往返时延远大于发送方的发送时延时（例卫星链路），信道利用率非常低，即分母RTT大

    解决方案：两种协议，后退N帧协议GBN和选择重传协议SR

    【2018年 题36】主机甲采用停-等协议向主机乙发送数据，数据传输速率为3kbps，单向传播时延是200ms，忽略确认帧的传输时延，当**信道利用率等于40%**时。数据帧的长度为（）

    A、240比特 B、400比特 C、480比特 D、800比特

    # 可靠传输的实现机制—回退N帧协议(GBN)

    1、采用3个比特给分组编序号，即0~7

    2、发送窗口的尺寸Wt的取值：1~7，本例取5，发送5个数据，Wt不能取8，因为可能会出现数据重复接收

    3、接收窗口的尺寸Wr的取值：Wr=1，即一个一个接收

    缺点：出现误码数据，其分组的其他数据也要重传

    累积确认：即使确认分组丢失，发送方也可能不必重传，因为接收方发送ACK1、ACK4，即使ACK1丢失，发送方也知道4之前的数据已接收

    有差错情况**：即5个数据当中有一个误码**，接收方就发送上次数据ACK（5个）到发送方，发送方收到重复（5个）的确认，就知道之前所发生的数据分组出现了差错，所以在本例中，即使前4个数据没错，但发送方还要重传这些数据，即Go-back-N

    【2009年 题35】数据链路层使用后退N帧（GBN）协议，发送方已经发送了编号为0~7的帧，当计时器超时时，若发送方只收到0、2、3号帧的确认，则发送方需要重发的帧数是（）

    A、2 B、3 C、4 D、5

    答案：C；收到3号帧，说明0~3都已接收，所以只需发送4、5、6、7即四个帧数

    # 可靠传输的实现机制 —— 选择重传协议（SR），常用

    发送方：发送窗口尺寸Wt范围是1<Wt<=2^(n-1)，n是分组序号的比特数量，范围超出，接收方无法分辨新、旧数据分组，一旦超出范围，即可能出现0、1和2

    接收方：

    1、Wr范围是1<Wr<=Wt，一般取Wt

    2、为了使发送方仅重传出现差错的分组，接收方不能再采用累积确认，而需要对每个正确接收到的数据分组进行逐一确认

    3、发送方和接收方只有当前窗口数据已接收或收到才能往下滑动

    【2011年 题35】数据链路层采用选择重传协议（SR）传输数据，发送方已发送了0~3号数据帧，现已收到1号帧的确认，而0、2号帧依次超时，则此时需要重传的帧数是（）

    A、1 B、2 C、3 D、4

    答案：B；选择重传协议不支持累计确认，收到1号帧的确认，说明还需要对0、2进行重传

    # 基本概念

    共享信道要考虑的问题是如何协调多个发送和接收站点对一个共享传输媒体的占用，即媒体接入控制MAC

    媒体接入控制包括静态划分信道和动态接入控制，而静态包括频分、时分、码分多址；动态包括受控、随机接入

    静态：预先固定分配好信道，不灵活，在无线网络的物理层

    动态：共享式以太网就是随机接入

    有线：点对点链路和链路层交换机的交换式局域网

    无线：共享媒体技术

    # 静态划分信道

    信道复用，即一条物理线路同时传输多路用户的信号

    信道复用包括4种

    频分复用（多址）FDM：所有用户同时占用不同的频带资源并行通信

    时分复用TDM：所有用户在不同的时间占用同样的频带宽度

    波分复用WDM：

    码分复用CDM：另一种共享信道的方法，

    1、用户可以在同样的时间使用同样的频带继续通信，用户之间不会造成干扰

    2、直接序列扩频DSSS：每一个比特时间再划分为m个短的间隔，称为码片，通常m的值是64或128

    每一个站被指派一个唯一的m bit码片序列，发送比特1，即发生自己的m bit码片序列，发送0，发自己的二进制反码

    例：CDMA系统中某个站点的码片序列为00011011，发送比特1还是00011011，而发送0是11100100，最后把1换为+1，0换为-1

    码片序列的挑选原则：

    1、每个站的码片序列必须各不相同

    2、每个站的码片序列必须相互正交（规格化内积为0）

    例：S·T=0，S·T的反码=0，S·S·=1，S·S的反码=-1

    不严格区分复用与多址的概念：复用是将单一媒体的频带资源划分成很多子信道，而多址是动态分配信道给用户

    # 动态接入控制 — 随机接入 — CSMA/CD（总线局域网）

    载波监听多址接入/碰撞检测 CSMA/CD

    1、多址接入MA：多个站连接在一条总线

    2、载波监听CS：每一个站发生帧之前检测总线是否有其他站点发送帧，即总线是否有96比特空闲

    3、碰撞检测CD：正在发送帧的站检测是否碰撞

    以太网采取一种叫做强化碰撞的措施，即碰撞除了立即停止还发送帧，以便其它站点知道总线繁忙

    CSMA/CD协议-争用期（碰撞窗口）

    主机最多经过2兀（端到端往返传播时延）时长知道本次是否发生碰撞，这个时长称为争用期或碰撞窗口，所以共享式以太网不能连接太多的主机

    CSMA/CD协议-最小和最大帧长

    最小帧长=争用期*数据传输速率

    **以太网最小帧长为64字节，**这确保碰撞后站点知道碰撞，然后重新发送，所以凡是长度小于64字节的帧都是碰撞后重新发送的帧

    以太网V2的MAC帧（最大长度为1518字节），802.1帧（最大长度为1522字节）

    CSMA/CD协议-截断二进制指数退避算法

    退避时间=基本退避时间（争用期2兀）* 随机数(2^k-1)，其中k=Min(重传次数，10)，好处是减小碰撞的概率

    CSMA/CD协议-信道利用率

    理想情况：极限信道利用率 S= T0/T0+兀，以太网帧的长度应尽量长些，端到端距离小点（兀）

    CSMA/CD协议-帧接收流程

    ==1、检测帧的长度==

    ==2、帧的目的MAC地址与接收方MAC地址是否相同==

    ==3、CRC检测帧是否出现误码==

    # 动态接入控制 — 随机接入 — CSMA/CA（无线局域网）

    载波监听多址接入/碰撞避免 CSMA/CA

    802.11的MAC层标准定义了两种不同的媒体接入控制方式：

    1、分布式协调功能DCF（争用信道来获取发送权）

    2、点协调功能PCF：使用集中控制的接入算法（很少用）

    帧间间隔IFS：

    所有的站点必须在持续检测到信道空闲一段指定时间后才能发送帧

    常用的两种帧间间隔：

    1、短帧间间隔SIFS（28us）：一个站点在这段时间内从发送方式切换到接收方式

    2、DCF帧间间隔DIFS（128us）：在DCF方式中用来发送==数据帧和管理帧==

    CSMA/CA协议的工作原理

    1、源站为什么在检测到信道空闲后还要再等待一段时间DIFS

    答案：考虑到高优先级帧，让它先发送

    2、目的站为什么正确接收数据帧后还要等待一段时间SIFS才能发送ACK帧？

    答案：用来分隔开属于一次对话的各帧

    3、信道由忙转为空闲且经过DIFS，为什么还要退避一段随机时间？

    答案：防止各个站点同时发送数据而碰撞

    必须使用退避算法：

    1、在发送数据帧之前检测到信道处于忙状态时；

    2、每一次重传一个数据帧时

    3、每一次成功发送后要连续发送下一个帧时

    退避原理：检测到信道忙，则进行退避，忙到空闲，退避时间继续计时，然而信道又忙了，此时冻结剩余退避时间

    CSMA/CA协议的信道预约和虚拟载波监听

    信道预约：源站先发送请求发送RTS，经过SIFS，目的站发送响应CTS，，经过SIFS，发送数据帧，RTS帧和CTS帧很短，用很小的代价对信道进行预约

    虚拟载波监听：除了RTS帧和CTS帧会携带通信需要持续的时间，数据帧也能携带，即携带需要用信道多久

    # MAC地址、IP地址以及ARP协议

    # MAC地址（物理地址、硬件地址）

    当多个主机连接在同一个广播信道上，每个主机发送的帧中必须携带标识发送主机和接收主机的地址

    MAC地址是对网络上各接口的唯一标识，而不是对网络上各设备的唯一标识，属于物理链路层

    IEEE 802局域网的MAC地址格式

    标准表示法：XX-XX-XX……（加-）

    其他表示法：XX:XX:XX……加（:）、XXXX.XXXX.XXXX……

    广播MAC地址，即每个主机都能接收，单播MAC地址，只能对应的主机接收，而多播MAC地址是该位十六进制不能整除2

    # IP地址（网络层）

    IP地址是因特网上的主机和路由器所构成的

    网络编号（即路由器）：标识因特网

    主机编号：标识同一网络上不同主机（路由器的接口，即连接一个路由器的多个主机）

    数据包转发过程中IP地址与MAC地址的变化情况：

    源IP地址和目的IP地址保存不变；而源MAC地址和目的MAC地址逐个链路改变

    # ARP协议

    即IP--获取MAC，例：两个主机知道各自的IP，当不知道各自的MAC，此时用ARP协议就可以查询

    主机B先发送ARP请求报文（广播），主机C发现是请求要自己的MAC地址，所以主机C把B的IP地址和自己的MAC地址记录到ARP高速缓存表中，最后发送单播

    动态：自动获取，生命周期默认为两分钟

    静态：手工设置，不同操作系统不一样

    # 集线器和交换机的区别

    集线器（淘汰）：

    1、使用集线器的以太网逻辑上还是一个总线网，使用的是CSMA/CD协议

    2、集线器只工作在物理层

    3、集线器有少量的容错能力和网络管理功能

    以太网交换机（流行）：

    1、有多个接口，无碰撞（不使用CSMA/CD）

    2、工作在数据链路层，收到帧后在帧交换表查找帧的目的MAC地址对应的接口号，其中帧交换表是通过自学习算法自动建立起来的

    3、帧的两种转发方式：存储转发（需要缓存），直通交换（交换时延很小，收到数据知道目的地址就转发，缺点是没检测帧是否有差错）

    对比集线器和交换机：

    **集线器扩大了广播域和碰撞域，而交换机只扩大了广播域，**两则的广播域没什么区别，交换机几乎没有碰撞，而且价格也很实惠。现在很流行

    # 虚拟局域网VLAN

    # 概述

    巨大的广播域弊端：

    1、==广播风暴，浪费网络资源和各主机的CPU资源==

    2、难以管理和维护

    IEEE 802.1Q帧：

    IEEE 802.1Q帧对以太网的MAC帧格式进行扩展，插入4字节的VLAN标记，其中标记的最后12比特称为VLAN标识符VID，范围是1~4094

    802.1帧是由交换机来处理的，而不是用户主机来处理，

    收到帧 “打标签”：插入4个字节

    转发帧 “去标签” 删除4个字节

    交换机的端口类型：

    交换机各端口的缺省：在华为中，VLAN ID ，简记PVID

    1、Access：

    交换机和用户计算机的互连

    Access端口只能属于一个VLAN

    发送处理方法：去标签，转发该帧

    接收处理方法：打标签，字段的VID取值与端口的PVID取值相等

    2、Trunk：

    交换机之间或交换机与路由器之间的互连

    Trunk端口可以属于多个VLAN

    发送处理方法：对VID等于PVID的帧，“去标签”再转发，不等于则直接转发

    发送处理方法（与Trunk端口不同）查看帧的VID是否在端口的“去标签”列表中：

    # IPv4地址及其应用

    # 概述

    IPv4地址是因特网给每一台主机（或路由器）的每一个接口分配一个在全世界范围内唯一的32比特的标识符

    IPv4地址的编址方法经历三个历史阶段：

    1、分类编址 2、划分子网 3、无分类编制

    # 分类编址的IPv4地址

    只有A、B、C类地址可分配给网络中的主机或路由器的各接口

    主机号为“全0”的地址是网络地址，主机号全为1的地址是广播地址，都不可以分配

    A类地址（0~127）：最大网络号127（01111111），作为本地环回测试地址，不指派

    可指派的网络数量：2^(8-1)-2=126，其中减2原因是除去最小网络号0和最大网络号127

    可分配的IP地址数量：2^(24)-2，减2原因是除去主机号为全0的网络地址和全1的广播地址

    B类地址（128~191）：最小网络号128.0，最大网络号191.255，注意：可指派网络数量包括最小和最大的网络号

    C类地址（192~223）：最小网络号192.00，最大网络号233.255.255，注意：可指派网络数量包括最小和最大的网络号

    例：

    IP 类别 是否可以指派
    0.1.2.3 A 不可以指派
    126.255.255.255 A 广播地址
    128.0.255.255 B 广播地址
    191.255.255.252 B 可以
    192.0.0.255 C 广播地址

    网络前缀最长，路由最具体，多条路由可选时，选最长前缀

    人工配置路由表可能会出现路由环路的错误：

    1、配置错误：即下一跳又跳回原来，解决方法：在IP数据报首部设有生存时间TTL字段

    2、聚合了不存在的网络：即**跳到不存在的地址，然后走默认地址，又回到原来，**解决方法：设置黑体路由（下一跳为null0）

    3、网络故障：和聚合了不存在的网络差不多，即跳到不存在的网络

    1、自适应：动态路由选择

    2、分布式：路由器之间交互路由信息

    3、分层次：将整个因特网分为许多较小的自治系统AS

    路由选择协议：

    1、内部网关协议IGP：

    基于距离向量：路由信息协议RIP和内部网关路由协议IGRP

    基于链路状态：开放式最短路径优先OSPF和中间系统到中间系统IS-IS（ISP骨干网常用的IGP协议）

    # 路由信息协议RIP的基本工作原理

    RIP使用跳数来衡量到达目的网络的距离，直连网络的距离定义为1，距离等于16相当于不可达

    好的路由：距离短，所通过路由器数量最少

    RIP包含以下三个要点：

    和谁交换信息 和相邻路由器交换信息

    交换什么信息 自己的路由表

    何时交换信息 周期性交换

    A的改造：都加一个1，下一跳都为A

    B的改造：添加N1，N2不变，N3为A中的，删除B中的N3，两个N5，N7更新为A的，因为下一跳相同，N9保存不变

    【2010年 题35】某自治系统采用RIP协议，若该自治系统内的路由器R1收到其邻居路由器R2的距离矢量，距离矢量中包含信息《net1，16》，则能得出的结论是（）

    A、R2可以经过R1到达net1，跳数为17

    B、R2可以到达net1，跳数为16

    C、R1可以经过R2到达net1，跳数为17

    D、R1不能经过R2到达net1

    答案：D，距离超过16说明不可达

    RIP存在“**坏消息传播得慢”**的问题

    解决方法：

    1、限制最大路径距离为15

    2、当路由表发生变化时就立即发送更新报文（触发更新），而不仅是周期

    3、不让同一路由信息再通过此接口向反方向传送（水平分割）

    # 开放最短路径优先OSPF的基本工作原理

    算法：最短路径算法SPF（Dijkstra）

    OSPF**基于链路状态，**RIP基于距离向量

    链路状态指本路由器和哪些路由器相邻，以及相应链路的代价

    例：思科路由器OSPF计算代价的方法：100Mbps/链路带宽，结果小于1的也记1

    OSPF相邻路由器之间通过交互问候分组，建立和维护邻居关系

    OSPF的每个路由器都会产生链路状态报告LSA，LSA包括直连网络和邻居路由器的链路状态信息，被封装在链路状态更新分组LSU中，采用洪泛法发送

    OSPF的每个路由器都有一个链路状态数据库LSDB，用于存储LSA，洪泛发送后，各路由器的LSDB最终一致

    其次，基于LSDB进行最短路径优先SPF计算，构建出各自路由表，即路由器到其他路由器的最短路径

    OSPF的五种分组类型：

    1、问候分组**：维护邻居路由器的可达性**

    2、数据库描述分组：给出自己的链路状态数据库

    3、链路状态请求分组：请求别的路由器链路状态信息，添加到自己中

    4、链路状态更新分组：被请求的路由器发送链路状态信息（洪泛法）

    5、链路状态确认分组：请求的接收到后，回应

    重点：为了使OSPF能够用于规模很大的网络，把一个自治系统划分为多个，即区域

    # 边界网关协议BGP的基本工作原理

    BGP只能是力求寻找一条能到目的网络且比较好的路由（不兜圈），并非是寻找最佳路径

    配置BGP：每个自治系统至少一个路由器作为该自治系统的BGP发言人

    不同自治系统的发言人：

    1、首先建立TCP连接

    2、在TCP连接上进行交互BGP报文以建立BGP会话

    3、利用会话交换路由信息

    4、最后发言人根据所采用的策略找到到达各自治系统较好的路由，即不存在回路的自治系统连通图

    BGP-4有以下四种报文：

    1、OPEN（打开）报文：与相邻的发言人建立关系

    2、UPDATE（更新）报文：通告某一路由的信息

    3、KEEPALIVE（保活）报文：周期性和相邻发言人连通

    4、NOTIFICATION（通知）报文：用来发送检测到的差错

    # IPv4数据报的首部格式

    1、版本：4个比特，表示IP协议的版本

    2、首部长度：占4比特，该字段取值以4字节为单位

    最小十进制为5，表示IP数据报首部只有20字节

    最大十进制15，表示首部包含20字节固定部分和最大40字节可变部分

    3、可选字段：开销大，用来支持排错

    4、填充字段：确保首部长度为4字节的整数倍，使用全0填充

    5、区分服务：占8比特，获得更好的服务

    6、总长度：占16比特，表示IP数据报的总长度（首部+数据载荷）

    7、标识：占16比特，属于同一个数据报的各分片数据报应该具有相同的标识

    8、标志：占3比特，各比特含义如下：DF位：1表示不允许分片；MF：1表示后面还有分片；保留位：必须为0

    9、片偏移：占13比特，指出分片数据报的数据载荷部分偏移其在原数据报有多少个单位，除于8

    10、生存时间TTL（防止兜圈）：占8比特，表示IP数据报的生存时间，到达路由器跳数-1，该字段-1后的值不为0则转发，否则丢弃

    11、协议：占8比特，说明是使用何种协议数据单元

    12、首部检验和：占16比特，检测首部在传输过程是否出现差错（和可选字段有区别），比CRC检验码简单，IPv6不再计算该字段

    # 差错报告报文

    五种：

    1、终点不可达：当路由器或主机不能交付数据报时（不知道如何转发），就向源点发送终点不可达报文

    2、源点抑制：由于拥塞而丢弃数据报

    3、时间超过：即生存时间TTL字段的值为0，就丢弃该IP数据报，并向源点发送时间超过报文；还有一种情况是不能按时到达终点，就把已收到的数据报片都丢弃，也会向源点发送时间超过报文

    4、参数问题：发现首部在传输过程中出现了误码

    5、改变路由（重定向）：可通过更好的路由

    两种：

    1、回送请求和回答：测试目的站是否可达

    2、时间戳请求和回答：从1900年1月1日开始到当前时刻一共多少秒，进行时钟同步和测量时间

    # 运输层的概述

    物理层、数据链路层以及网络层共同解决了主机通过异构网络互联起来的问题，实现了主机到主机的通信

    # TCP的拥塞控制

    拥塞：若对网络中某一资源的需求超过了该资源所能提供的可用部分，网络性能就变坏，网络资源：带宽、交换结点中的缓存和处理机等

    若出现拥塞不进行控制，则网络的吞吐量将随输入负荷的增大而下降，吞吐量：在单位时间内通过某个网络的数据量

    解决方法：

    发送方维护一个叫拥塞窗口cwnd的状态变量，取决于网络的拥塞程度；一个慢开始门限ssthresh，当cwnd小于ssthresh时，使用慢开始算法，大于使用拥塞避免算法

    1、慢开始：指一开始向网络注入的报文段少； 从1开始

    2、拥塞避免：并非能完全避免拥塞，只是将拥塞窗口控制为按线性规律增长

    个别报文段在网络中丢失，网络并未发生拥塞，发送方误认为拥塞，然后超时重传，降低传输效率

    3、快重传（新的）：

    ·接收方要立即发送确认，不要等待自己发送数据时才进行捎带确认

    ·即使出错的报文段，也要立即发出对已收到的报文段的重复确认

    ·发送方一旦收到3个连续的重复确认，相应的报文段立即重传，不是等待超时重传

    4、快恢复（新出的）：发送方将慢开始门限值和拥塞窗口值调整为当前窗口的一半，开始执行拥塞避免算法，没有慢开始算法，从中间值开始

    TCP客户 TCP服务器
    主动关闭，发送TCP连接释放（进入FIN-WAIT-1终止等待1） 被动关闭，发送TCP普通确认（CLOSE-WAIT关闭等待）

    # TCP报文段的首都格式

    TCP在发送数据时，添加一个首部使之成为TCP报文段

    TCP报文段由首部和数据载荷两部分构成

    TCP的全部功能都体现在首部中各字段作用

    1、序号：指出TCP报文段数据载荷的第一个字节的序号

    2、确认号：例若确认号=n，表示n-1之前的所有数据已正确接收，期望接收序号从n开始的数据

    3、确认标志位ACK：在连接建立后所有传送的TCP报文段都必须把ACK的值设为1

    4、数据偏移：占4比特，以4字节为单位，指出数据载荷部分开始处距离TCP报文段的开始处有多远，差一个首部距离，即计算的时首部的长度

    首部最大长度，11114=15 * 4=60，最小：01014=5 * 4=20

    5、窗口：占16比特，接收窗口的大小

    6、校验和：占16比特，检查范围包括首部和数值载荷，校验时加上12字节的伪首部

    7、复位标志位RST：复位TCP连接，RST=1：连接异常，释放后重新连接；还有一种可能是拒绝非法的报文段或打开一个TCP连接

    8、推送标志位PSH：值为1，尽快上交应用进程，不必等到接收缓存填满再上交

    9、紧急标志位URG：值为1开启，当发送方有紧急数据时，把紧急数据插队到发送缓存最前面，并立即发送

    10、紧急指针：占16比特，用来指明紧急数据的长度，而紧急数据之后是普通数据

    11、选项：最大报文段长度MSS（数据载荷最大长度）；窗口扩大选项（扩大窗口，提高吞吐率）；时间戳（计算往返时间）

    # 动态主机配置协议DHCP

    通过DHCP自动获取网络配置信息，即插即用连网，允许一台计算机加入新网络时可自动获取IP地址等网络配置信息而不用手工参与

    DHCP的工作过程：

    运输层使用UDP协议封装：DHCP客户使用的UDP端口号为68，服务器为67，DCHP客户未获取到IP地址时使用地址0.0.0.0

    1、寻找DHCP服务器（DISCOVER）：广播

    2、服务器提供IP地址租用（OFFER）：广播发送，服务器使用ARP确保所选IP地址未被网络中其他主机使用

    3、接受IP地址租约（REQUEST）：广播，其中包括事务ID，DHCP客户端的MAC地址，接受租约中的IP地址，提供此租约的DHCP服务器端的IP地址

    4、服务器确认IP地址租约（ACK）：广播，客户使用ARP检查IP地址是否被其他主机使用

    5、IP地址续约：到0.5租用期时，客户发送续约请求，同意则继续使用，不同意则发送DISCOVER（寻找）

    6、随时解除IP地址租约（RELEASE）：客户立即停止使用

    DHCP中继代理：给路由器配置DHCP服务器的IP地址，使之成为DCHP中继代理，作为各网络中计算机与DHCP服务器的桥梁，减少DHCP服务器的数量

    # 域名系统DNS

    因特网采用层次结构的命名树作为主机的名字（即域名），DNS大多数域名都在本地解析，系统效率高，DNS是分布式系统，单个计算机出了故障不影响其他

    层次树状结构的域名结构：

    各分量之间用**“点”隔开**，代表不同级别的域名，级别最高的顶级域名写在最右边

    顶级域名TLD分为三类：

    1、国家顶级域名nTLD：采用ISO 3166的规定，如：cn表示中国，us美国

    2、通用顶级域名gTLD：最常见的通用有7个，即：com（公司）、net（网络服务机构）、org（非营利性组织）、int（国际组织）、edu（美国教育机构）、gov（美国政府部门）、mil（美国军事部门）

    3、反向域aepq：即IP地址反向解析为域名

    国家顶级域名下注册的二级域名均由该国家自行确定，我国二级域名划分为两类：类别域名和行政区域名，二级和顶级可能相同，不过没影响

    这种按等级管理的命名方法便于名字的唯一性、也容易设计查找方法

    DNS使用分布在各地的域名服务器来实现域名到IP地址的转换

    域名服务器划分为以下四种类型：

    1、根域名服务器（代理）：服务器群集，根域名服务器通常并不直接对域名进行解析，而是返回该域名所属顶级域名服务器的IP地址

    2、顶级域名服务器：管理在该顶级域名服务器注册的所有二级域名，返回结果可能是（需要的，也可能是下一级权限域名服务器的IP地址）

    3、权限域名服务器：管理某个区的域名，每一个主机的域名都必须在某个权限域名服务器注册登记，知道域名与IP地址的映射关系

    4、本地域名服务器（代理）：报文首先送到该主机的本地域名服务器，本地域名服务器的IP地址需要直接配置在需要域名解析的主机中

    域名解析的过程：递归查询和迭代查询

    通常采用以下模式：从请求主机到本地域名服务器的查询是递归查询，而其余查询是迭代

    为了提高DNS的查询效率：域名服务器使用了高速缓存，存放最近查询过的域名以及从何处获得域名映射信息的记录

    负担重 如果是流水线，那么后序请求只消耗一个RTT，不管多少个请求，而非流水线，一个请求消耗一个RTT
    流水线：一个请求还没到，另一个请求接着发；而非流水是等请求到再发另一个

    HTTP的报文格式：面向文本，其中每一个字段都是一些==ASCII码串==

    使用Cookie记录用户信息：是一种对==无状态==（静态文档）的HTTP转变为==状态化==的技术

    原理：

    1、服务器为客户生成一个==唯一的Cooked识别码==，记录该用户==访问该网站的各种信息==

    2、下次访问时，==服务器根据Cookie识别码==就可以识别出该用户，并返回该用户的个性化网页

    万维网缓存与代理服务器：

    ==万维网缓存又称为Web缓存==，可位于客户机，也可位于中间系统上，位于中间系统也叫代理服务器

    缓存把最近一些请求和响应存在本地磁盘中，请求是和缓存相同的，就返回该响应，不用URL去因特网访问资源
