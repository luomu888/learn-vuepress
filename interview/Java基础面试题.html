    # 简单说下什么是跨平台

    操作系统的指令集，屏蔽系统之间的差异，（就是在操作系统之间加个虚拟机，提供统一的接口）

    byte、short、int、long、float、double、char、boolean

    # 面向对象特征

    封装、继承、多态、抽象

    封装：把一个物体的行为和、特征包装起来

    把对象的属性和行为封装在一个类中，属性用变量来定义，行为用方法定义，方法可以直接访问在同一个对象中的属性

    抽象：把一个物体的行为和、特征抽象为方法和属性

    过程抽象：把鸟在飞翔（类的方法）

    数据抽象：鸟的羽毛（类的属性）

    继承：子类继承父类、可以拥有父类的方法和属性，属性（非private）。并且可以重写父类的方法，还可以添加自己独特的方法，但是耦合性很高

    多态：主要体现在重载和覆盖，在书写时，引用指向的类型、方法的调用类型不清楚，在运行时才知道

    方法的覆盖（重写）是子类和父类之间的关系，是垂直关系；方法的重载是同一个类中方法之间的关系，是水平关系。

    常见的包装类型：

    byte----->Byte

    为什么要有：因为在集合里我们放的是对象，包装类型可以让它变成对象，容器装的是object

    在集合里放的是包装类型，不是基本类型，容器装的是object

    自动装箱：new Integer 底层调用Integer.valueof(6)

    自动拆箱：int i=new Integer(6) 底层调用i.intvalue(6)

    # 二者的区别：

    Integer i=6;

    声明方式：底层实现原理：new Integer(6)

    存储位置：基本类型放在栈中，包装类型放在堆中（需要new）,通过引用来调用

    # ==和equals区别

    String底层equals已经重写了（源码：String重写的equals不仅有内存地址的比较，还增加字符串长度的是否相同）

    ==一般用来比较引用指向对象是否相等（即内存空间在存储的位置是否一致），equals比较的是特征，也就是内容之间是否相等

    # String、StringBuffer和StringBuilder区别

    StringBuilder：是线程不安全的，效率快，

    StringBuffer(字符串缓冲区)：线程安全，同步

    都继承了AbstractStringBuilder

    StringBuilder、StringBuffer：char[] value

    数据可变与不可变

    String: private final char value[] 底层使用**不可变的字符数组，**所以内容不可变

    StringBuilder、StringBuffer都继承了AbstractStringBuilder,可变字符数组，char[] value

    # ArrayList和LinkedList区别？

    ArrayList实现了基于动态数组的数据结构，LinkedList实现了基于链表的数据结构

    对于随机访问的get和set ArrayList好，因为LinkedList要移动指针

    对于数据的新增和删除 LinkedList好，因为ArrayList还要数据的移动

    # ConcurrentModificationException异常出现的原因

    # HashMap和HashTable、ConcurrentHashMap区别？

    相同点：都实现了Map接口，都可以存放key-value

    HaspMap:线程不安全,效率快，key-value 可以存放null，迭代器（iterator）：是fail-fast迭代器（最快把错误抛出，停止运行，）

    HashTable:线程安全 ，不可以存放null，迭代器（enumerator）

    CouncurrentHashMap：替代HashTable，把Map分成N个HashTable(N默认16)

    HashMap可以变成同步的

    # 线程池创建线程（少用）

    public class CreatThreadDemo6 {
    public static void main(String[] args) {
        //创建一个具有10个线程的线程池
        ExecutorService threadPool = Executors.newFixedThreadPool(10);
        long threadpoolUseTime = System.currentTimeMillis();
        for (int i = 0;i<10;i++){
            threadPool.execute(new Runnable() {
                @Override
                public void run() {
                    System.out.println(Thread.currentThread().getName()+"线程执行了...");
                }
            });
        }
        long threadpoolUseTime1 = System.currentTimeMillis();
        System.out.println("多线程用时"+(threadpoolUseTime1-threadpoolUseTime));
        //销毁线程池
        threadPool.shutdown();
    }

}

    # 面向对象六大原则

    接单依赖迪里开

    接口隔离(ISP)：把接口分成满足依赖关系的最小接口，实现类不能有不需要的方法

    单一职责(SRP)：每个类只专注与自己的方法

    依赖倒置(DIP)：调用的是接口，不是具体的实现类

    里氏替换(LSP)：重写父类，扩展性高，但不可改变原来父类拥有的方法

    开闭原则（OCP）：对象中的（类，函数，模块）对外可扩展，对修改关闭

    迪米特(LOD)：高内聚，低耦合

    高内聚：简单来说就是单一职责

    # static和final区别

    final：最终的，不可变

    1、放在常量池中，程序不可修改

    2、不可以被继承

    3、子类不能被重写

    static：静态的

    1、分配在堆，只分配一次内存

    2、直接类点来调用，不用new

    3、方法块，虚拟机优先加载

    # String s = "hello"和String s = new String("hello");区别

    String s="hello" ：首先查找常量池里有没有，没有则分配（只是分配，没有new）空间，（过程不涉及new 堆）

    String s=new String(:hello) ：先new 在找，没有在new （可能new 两次,这个常量池没有要new）
