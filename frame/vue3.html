    Vue3快速上手

    # Vue3快速上手

    # Vue3简介

    ==2020年9月18日==，Vue.js发布3.0版本，代号：One Piece（海贼王）

    • GitHub上的tags地址：https://github.com/vuejs/vue-next/releases/tag/v3.0.0

    # Vue3的不同之处

    # 性能的提升

    • 打包大小==减少==41%，内存减少54%
    • 初次渲染==快==55%，更新渲染快133%

    # 源码的升级（重点）

    • ==Proxy代替defineProperty实现响应式==
    • ==重写虚拟DOM和Tree-Shaking==（树在抖动，术语）:自动删除未引用的代码，具有简洁的结构和最小化大小。

    # 拥抱TypeScript

    • Vue3更好的支持TS，==以后主流TS，不用写js==

    # 新的特性（重点）

    1. Composition API（组合API）
      • setup配置
      • ref与reactive
      • watch与watchEffect
      • provide与inject
    2. 新的内置组件
      • Fragment
      • Teleport
      • Suspense
    3. 其他（很小的改变）
      • 新的生命周期钩子
      • data只能为一个函数
      • 移除==keyCode==支持作为v-on的修饰符：==即不支持@key-up.13，可以@key-up.enter==

    # 一、创建Vue3.0工程

    # 使用vue-cli创建

    官方文档：https://cli.vuejs.org/zh/guide/creating-a-project.html#vue-create

    • ==查看当前@vue/cli版本==，确保@vue/cli版本在4.5.0以上：==vue -V==
    • 安装或升级@vue/cli：==npm install -g @vue/cli==
    • 创建：==vue create 名字==
    • 启动：先切换进去，然后==npm run serve==

    # 使用vite创建

    官方文档：https://v3.cn.vuejs.org/guide/installation.html#vite

    vite官网：https://vitejs.cn

    • vite：==新一代前端构建工具，之前是webpack==

    • 优势：

      • 开发环境，==无需打包操作，可快速的冷启动==

      • 轻量==快速的热重载==（HMR）

      • ==按需编译==，不需等待整个应用编译完成

      • 传统：需要加载完，服务器才开始准备

      • 现在：服务器已准备好了

    • 使用教程：

      • 创建：==npm init vite-app 名字==
      • 进入工程：==cd 名字==
      • ==npm i==：安装node_modules
      • 运行：==npm run dev==(注意，这和之前不一样)

    # 二、常用 Composition API

    # 拉开序幕的setup

    1. 新的配置项，==代替data，值为一个函数==

    2. setup是==所有组合（Composition）API的基础，即表演的舞台==

    3. 数据、方法等都要配置在这

    4. setup函数的==返回值：把属性、方法暴露出去，让模板可以直接使用==

    5. 注意：

      1. 不要和vue2配置一起使用
        • ==vue2可以访问到vue3配置==
        • ==vue3不可以==访问到vue2配置

      2. setup不能是async（异步）函数，因为返回值不是return，而是promise，模板看不到return（==后期可以返回Promise实例，但需要Suspense和异步组件的配合）==

      //vue3的main.js
import { createApp } from 'vue'
import App from './App.vue'

createApp(App).mount('#app')

//vue2的main.js
new Vue({
  el: '#app',
render: h => h(App)
});

    # ref函数

    定义一个响应式的数据，常用来==定义基本类型数据==

    • 语法：==const xxx=ref(初始值)==

      • 使用之前要引入：==import {reactive,ref} from 'vue'==
      • 修改数据时：要获取它的value改，==sex.value='女'==
      • 模板读取数据时不需要加value
      • 底层靠的是object.defineProperty()的get和set完成的

    # reactive函数

    定义一个==对象类型的响应式数据（==基本类型使用ref，数组、对象使用reactive）

    • 语法：==const 对象=reactive({})，返回一个代理对象，简称proxy对象==
    • ==内部基于ES6的Proxy实现==，通过代理对象操作源对象内部数据进行操作
    	const person = reactive({
        // 基本数据
        name:'张三',
        age:18,
        // 对象数据
        job:{
          work:'吃饭'
        },
        // 数组数据
        hobby:['抽烟','喝酒','烫头']
      })

    # vue2的响应式

    • 实现原理：

      • 对象类型：通过==object.defineProperty()==

      • 数组：==重写数组的一系列方法（对数组的变更方法进行了包裹）==

        Object.defineProperty(data, 'count', {
    get () {}, 
    set () {}
})

    • 存在问题：

      • 增删，界面不会更新，即没有双向数据绑定
      • 直接通过下标修改数组，界面不更新，而vue3可以

    # vue3的响应式

    • 实现原理：

      • 通过==Proxy（代理）==

      • 通过==Reflect（反射）：真正操作数据的==

      • MDN文档中描述Proxy与Reflect

        • Proxy：https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Proxy

        • Reflect：https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Global_Objects/Reflect

          new Proxy(data, {
	// 拦截读取属性值
    get (target, prop) {
    	return Reflect.get(target, prop)
    },
    // 拦截设置属性值或添加新属性
    set (target, prop, value) {
    	return Reflect.set(target, prop, value)
    },
    // 拦截删除属性
    deleteProperty (target, prop) {
    	return Reflect.deleteProperty(target, prop)
    }
})

proxy.name = 'tom'

    # reactive对比ref

    • 从==定义==角度：
      • ref定义：==基本类型数据==
      • reactive定义：==对象或数组类型数据==
      • 备注：ref也可以定义对象或数组类型数据，它内部会自动通过reactive转为代理对象
    • 从==原理==角度：
      • ref通过==object.defineProperty()的get和set实现响应式（数据劫持）==
      • reactive通过使用==Proxy实现响应式（数据劫持），并通过Reflect操作源对象内部的数据==
    • 从==使用==角度：
      • ref数据：==操作需要.value==，读取不需要
      • reactive数据：==操作和读取不需要操作.value==

    # setup的两个注意点

    • 执行时机：

      • 在beforeCreate之前执行，即this是undefined

    • setup的参数：

      • ==props==：值为对象，==组件外部传递过来的，内部声明接收的属性==，直接proxy.address即可拿到数据

      • ==context==：上下文对象

        • attrs：组件外部传过来，但内部==没有声明props接收，就放到attrs，即捡漏==

        • ==slots：收到插槽的内容==

        • ==emit：自定义事件==

    APP.vue
<template>
  <!-- slot插槽，是写在组件里，组件是父里面写的子 -->
  <Demo @hello='showInfoHello' address='luo' sex='男'>
    <template v-slot:asd>
      <span>你好呀</span>
    </template>
    <template v-slot:asdf>
      <span>你好呀</span>
    </template>
  </Demo>
  
</template>

<script>
  import Demo from './components/Demo.vue'
  export default {
    name: 'App',
    components:{
      Demo
    },
    setup() {
      // 自定义事件hello，查看此事件的参数值
      function showInfoHello(value) {
        console.log(value);
      }
      return{
        showInfoHello
      }
    }
    
  }
</script>

    # computed函数

    • 简写：computed(==()=>{}==)
    • ==完整==写法：computed(=={}==)
    <template>
  <h1>一个人的信息</h1>
  姓：<input type="text" v-model="person.firstName">
  <br>
  名：<input type="text" v-model="person.lastName">
  <br>
  全名：<span>{{person.fullName}}</span>
  <br>
  全名：<input type="text" v-model="person.fullName">
  全名：<input type="text" v-model="person.Name">
  <br>
</template>

<script>
    //注意，这里是从vue里引入reactive
    import {computed,reactive} from 'vue'
    export default {
        name:'Demo',
        setup() {
            // 想要数据轻松快速双向绑定，使用reactive
            const person=reactive({
                firstName:'luo',
                lastName:'mu'
            })
            // 简写形式
            person.Name=computed(()=>{
                return person.firstName+'-'+person.lastName
            })
            // 注意这里要写computed({})，完整写法
            person.fullName=computed({
                // 获取数据
                get(){
                    return person.firstName+'-'+person.lastName
                },
                // 修改数据
                set(value){
                    const nameArr=value.split('-');
                    person.firstName=nameArr[0];
                    person.lastName=nameArr[1];
                }
            })
            return {
                person
            }
        }
    }
</script>

    # watchEffect函数

    • ==watch==：==声明监视的属性==，也要==指明监视的回调==

      • 回调可以写箭头函数，因为setup里this是undefined，setup创建比this创建的早
      • watch监视reactive定义的响应式数据时（对象），oldValue无法获取，强制开启了深度监视（deep失效，关不了）
      • watch监视reactive定义的==响应式数据中某个属性==：deep配置有效
      	// 第一个参数是绑定那个要监视，第二个是回调，第三个是配置
      watch(sum,(newValue,oldValue)=>{
        console.log(newValue,oldValue);
      },{immediate:true})

	//单个属性
	()=>person.job
	//多个属性，用数组
	[()=>person.job,()=>person.name]

    • ==watchEffect==：不用指明监视的属性，==只要回调中用到哪个属性，就监视哪个属性==

    • watchEffect有点像computed：

      • 但computed==注重计算==（回调的返回值），==必须写返回值==
      • 而watchEffect==注重过程==（回调的函数体），==不用写返回值==
    <template>
    <h2>当前求和为：{{sum}}</h2>
    <button @click="sum++">点我+1</button>
    <hr>
    <h2>当前的信息为：{{msg}}</h2>
    <!-- 注意：+='-' -->
    <button @click="msg+='-'">修改信息</button>
    <hr>
    <h2>姓名：{{person.name}}</h2>
    <h2>年龄：{{person.age}}</h2>
    <h2>薪资：{{person.job.j1.salary}}K</h2>
    <button @click="person.name+='~'">修改姓名</button>
    <button @click="person.age++">增长年龄</button>
    <button @click="person.job.j1.salary++">涨薪</button>
</template>

<script>
    //注意，这里是从vue里引入reactive，一般用到哪个方法就要从vue引入
    import {ref,reactive,watchEffect} from 'vue'
    export default {
        name:'Demo',
        setup() {
            // 定义和
            const sum=ref(0)
            const msg=ref('你好啊')
            const person = reactive({
                name:'luo',
                age:21,
                job:{
                    j1:{
                        salary:10
                    }
                }
            })
            watchEffect(()=>{
                // 注意：这里写了但没使用，会报错，需要修改vue.config.js的配置
                const x1=person.name
                const x2=person.age
                console.log("name或age被修改了");
            })
            return {
                person,
                sum,
                msg
            }
        }
    }
</script>

    # toRef和toRefs

    创建ref对象，其value值指向另一个对象（reactive定义的）中的某个属性

    • 语法：

      • toRef：==salary:toRef(person.job,j1,'salary')==
      • ==toRefs(person)==：对第一层有用

    • 两个可以联合使用

      <script>
    //注意，这里是从vue里引入reactive，一般用到哪个方法就要从vue引入
    import {reactive, toRef, toRefs} from 'vue'
    export default {
        name:'Demo',
        setup() {
            const person = reactive({
                name:'luo',
                age:21,
                job:{
                    j1:{
                        salary:10
                    }
                }
            })
            return {
                person,
                ...toRefs(person),
                salary:toRef(person.job.j1,'salary')
            }
        }
    }
</script>
