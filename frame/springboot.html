    # SpringBoot 概述

    # 概念

    Spring Boot 是一种==快速使用Spring的方式==，功能和Spring差不多，==并没有增强==

    • 快速使用Spring的方式，基于==约定优于配置==的思想
    • 约定：即spring配置都配好了

    # Spring 缺点

    • 配置繁琐：配置和解决业务问题之间思维切换
    • 依赖繁琐：防止版本冲突

    # SpringBoot 功能

    • 自动配置：应用程序启动时，==根据需求自动配置（即ssm的配置）==
    • 起步依赖：Maven项目对象模型，==相当于引jar包（pom.xml）==，不过这些包互相关联，引一个其它自动引进来，不需要考虑版本冲突
    • 辅助功能：==嵌入式服务器（不需要配置tomcat）==、安全、健康检测等

    # SpringBoot 快速入门

    1. 创建Controller包，编写Controller
    2. 编写引导类，即main方法（项目入口）
    3. 开始运行

    注意：==Spring Boot和Spring的项目，业务代码编写方式完全一样==

    官网文档：https://docs.spring.io/spring-boot/docs/current/reference/htmlsingle/#getting-started.first-application

    底层怎么知道我有没有导入依赖（坐标），然后创建哪个Bean（对象）

        	ConfigurableApplicationContext context = SpringApplication.run(SpringbootConditionApplication.class, args);
	//getBean获取Bean，即对象

      2. 创建config（配置包）

        3.使用注解

        ConditionalOnClass底层源码

    • 自定义注解实现以上功能（==不常用==）

      1. ==创建注解（头部引用底层的头部）==

      2. 自定义类==实现Condition接口==，重写matches方法，方法返回值是boolean，==返回true则识别Bean==，所以在这个方法中进行逻辑判断。该方法有两个参数：

        • context：上下文对象，获取BeanFactory（Bean的工厂）
        • metadata：==元数据对象，获取注解属性==

        Class.==forName==(className)：判断是否有这个类，没有则异常

        ==getAnnotationAttributes==：获取注解的属性值

        @Override
    public boolean matches(ConditionContext context, AnnotatedTypeMetadata metadata) {
        //标识是否可创建
        boolean flag = true;
        //获取注解的属性值
        Map<String, Object> map = metadata.getAnnotationAttributes(ConditionOnClass.class.getName());
        //通过ket获取值
        String[] value = (String[]) map.get("value");

        try {
            //遍历字符串数组，拿到类名进行判断
            for (String className : value) {
                Class<?> cls = Class.forName(className);
            }
        } catch (ClassNotFoundException e) {
            flag = false;
        }
        return flag;
    }

      3. 记得==加注解Configuration（知道是配置config）和Bean（知道是Bean，即对象），==

        @Configuration
public class UserConfig {

    @Bean
    @ConditionOnClass("redis.clients.jedis.Jedis")
    public User user(){
        return new User();
    }

      • ConditionalOn==MissingBean==：==没有对应的Bean==才初始化Bean

    # @EnableAutoConfiguration注解（整合前面几种Bean的创建，重点）

    ==描述底层怎么自动配置==

    • 注解内部使用@Import(AutoConfigurationImportSelector.class)来加载配置类
    • 配置文件位置：META-INF/spring.factories，当SpringBoot应用启动时，会自动加载这些配置类，初始化Bean
    • 不是所有的Bean都会初始化，使用Condition来加载满足条件的Bean

    # 案例：自定义redis-starter。要求当导入redis-starter坐标时，SpringBoot自动创建Jedis的Bean

    通过源码，我们可知，在自动配置里要创建配置文件，然后让s==pringBoot识别配置文件的位置（META-INF/spring.factories），也需要导入Jedis的坐标，即依赖==

    1.自动配置文件
	//标识是配置类
    @Configuration
    //指定配置属性的位置，让Spring识别到放到容器IOC里
    @EnableConfigurationProperties(JedisProperties.class)
    //有Jedis这个字节码文件（即导入依赖）才会执行下面自动创建Bean的过程
    @ConditionalOnClass(Jedis.class)
    public class JedisAutoConfiguration {

        /**
         * 创建Jedis的Bean
         * @param jedisProperties
         * @return
         */
        @Bean
        //不存在这个Bean才用我们自己创建的Bean
        @ConditionalOnMissingBean(name = "jedis")
        public Jedis jedis(JedisProperties jedisProperties){
            //动态指定主机和端口号
            return new Jedis(jedisProperties.getHost(),jedisProperties.getPort());
        }
    }
		//动态指定主机号和端口号的实体类
		@ConfigurationProperties(prefix = "redis")
    public class JedisProperties {
        private String host="localhost";
        private int port=6379;

        public String getHost() {
            return host;
        }

        public void setHost(String host) {
            this.host = host;
        }

        public int getPort() {
            return port;
        }

        public void setPort(int port) {
            this.port = port;
        }
    }
2.导入Jedis的依赖
		<dependency>
			<groupId>redis.clients</groupId>
			<artifactId>jedis</artifactId>
		</dependency>

    # SpringBoot 监听机制

    对Java提供的事件监听机制的封装，底层还是使用Java

    • 事件：Event
    • 事件源：Source
    • 监听器：Listener

    SpringBoot项目启动时，会==对几个监听器进行回调==，可以==实现这些监听器接口(已实现方法的接口)==：

    • ApplicationContextInitializer（在spring.factories里配置）：项目开始前，检测一些资源是否存在
    • SpringApplicationRunListener（在spring.factories里配置，==对应的接口全限定名称=自己实现接口的全限定名称==）：实现六个方法，跟生命周期差不多
    • CommandLineRunner和ApplicationRunner：用一个就行，功能一样，只是参数不同，完成缓存预加载，redis经常这样干
